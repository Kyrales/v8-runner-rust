# OIDA

- Вид: `parameter`
- Группа: Настройка аутентификации
- Source pagePath: `zif2_oida`
- Source URL: `http://localhost:8080/ru/1%D0%A1%3A%D0%9F%D1%80%D0%B5%D0%B4%D0%BF%D1%80%D0%B8%D1%8F%D1%82%D0%B8%D0%B5?page=zif2_oida`

## Синтаксис

```text
/OIDA[+/-]
```

## Нормализованное описание

— применение сквозной аутентификации пользователя между разными информационными базами и/или внешними ресурсами. (Используется только для **тонкого и веб-клиентов**!):

**/OIDA+** — использовать OpenID-аутентификацию (по умолчанию).

**/OIDA-** — не использовать OpenID-аутентификацию.

Если при запуске клиента параметр OIDA не задан, или задан параметр OIDA+, то производится попытка аутентификации через OpenID-провайдера, адрес которого задан в файле default.vrd публикации этой информационной базы.

Если OpenID-провайдер требует интерактивной аутентификации (происходит первое обращение или истекло время жизни признака аутентифицированности), клиент отображает диалог для ввода имени и пароля пользователя.

Аутентификация происходит по списку пользователей информационной базы OpenID-провайдера.

Аутентифицируемый пользователь информационной базы, использующей OpenID-аутентификацию, должен соответствовать по имени пользователю информационной базы OpenID-провайдера.
