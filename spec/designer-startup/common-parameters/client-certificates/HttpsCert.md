# HttpsCert

- Вид: `parameter`
- Группа: Использование клиентских сертификатов (только для тонкого клиента)
- Source pagePath: `zif2_httpscert`
- Source URL: `http://localhost:8080/ru/1%D0%A1%3A%D0%9F%D1%80%D0%B5%D0%B4%D0%BF%D1%80%D0%B8%D1%8F%D1%82%D0%B8%D0%B5?page=zif2_httpscert`

## Синтаксис

```text
/HttpsCert [-windows] [-linux] [-macos] [-recent] [-auto] [-choose] [-file <path>] [-pwd <password>] [-none]
```

## Нормализованное описание

— указывает источник клиентского сертификата. Если данный ключ не указан, то источник клиентского сертификата определяется в диалоге настройки соединения с информационной базой.  (Используется только в **тонком** клиенте!)

**-windows** — указывает, что при соединении нужно использовать клиентский сертификат из системного хранилища сертификатов операционной системы Microsoft Windows. Данный параметр игнорируется, если установлен хотя бы один из следующих параметров ключа: **-file** или **–none**.

**-linux** — указывает, что для проверки сертификата сервера при соединении нужно использовать сертификаты удостоверяющих центров из специального каталога ОС Linux, в котором хранятся сертификаты. Данный параметр игнорируется, если установлен хотя бы один из параметров ключа **-file** или **-none**.

**-macos** — указывает, что для проверки сертификата сервера при соединении нужно использовать сертификаты удостоверяющих центров из системного хранилища сертификатов ОС macOS. Данный параметр игнорируется, если установлен хотя бы один из параметров ключа **-file** или **-none**.

**-recent** — выбирать или использовать ранее выбранный клиентский системный сертификат Microsoft Windows или macOS.

Если в системном хранилище пользовательских сертификатов присутствует более одного подходящего сертификата, то пользователю предоставляется выбор используемого сертификата через системный диалог выбора сертификата. В дальнейшем выбранный сертификат используется автоматически.

Данный способ выбора клиентского сертификата выбран по умолчанию для параметра ключа **-windows **и** -macos**, если параметры **-auto** и **-choose** не установлены.

**-auto** — использовать автоматически выбранный клиентский сертификат из установленных в системном хранилище сертификатов операционной системы Microsoft Windows или macOS. Данный параметр игнорируется, если у параметра отсутствует параметр **–windows** или** -macos**.

**-choose** — всегда выбирать используемый клиентский сертификат Microsoft Windows или macOS.

Если в системном хранилище пользовательских сертификатов присутствует более одного подходящего сертификата, то пользователю предоставляется выбор используемого сертификата через системный диалог выбора сертификата независимо от того, был ли ранее выбран какой-либо сертификат. Выбранный сертификат в дальнейшем может использоваться автоматически с ключом **–recent**.

Данный параметр можно указать, если необходимо избежать автоматического использования ранее выбранного клиентского сертификата из системного хранилища сертификатов операционной системы Microsoft Windows, и выбрать новый сертификат из установленных в системе сертификатов подходящих для данного соединения. Данный параметр игнорируется, если у ключа отсутствует параметр **-windows/-macos** или установлен параметр **-auto**.

**-file <path>** — указывает, что необходимо использовать клиентский сертификат и приватный ключ из указанного файла. Данный параметр игнорируется, если у ключа установлен параметр **-none**.

**-pwd <password>** — указывает пароль файла, содержащего клиентский сертификат и его приватный ключ. Если сервер требует предоставления клиентского сертификата и файл сертификата защищен паролем, то соединение возможно только при правильно указанном пароле. Данный параметр игнорируется, если у данного ключа не указан параметр **-file**.

**-none** — указывает, что клиентский сертификат не используется. Соединение возможно только с серверами, не требующими проверки клиентского сертификата.

Если ни один из параметров **-windows**, **-linux**, **-macos**, **-file** или **-none** не установлен, то ключ **/HttpsCert** игнорируется.
