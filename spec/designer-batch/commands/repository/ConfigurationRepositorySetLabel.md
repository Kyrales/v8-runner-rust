## Синтаксис

```text
/ConfigurationRepositorySetLabel [-Extension <имя расширения>]
[-v <номер версии хранилища>] [-name"<название метки>"] [-comment"<текст комментария>"]
```

## Нормализованное описание

— установка метки для версии хранилища. Допустимо использование следующих параметров:

**-Extension <имя расширения>** — имя расширения. Если параметр не указан, выполняется попытка соединения с хранилищем основной конфигурации, и команда выполняется для основной конфигурации. Если параметр указан, выполняется попытка соединения с хранилищем указанного расширения, и команда выполняется для этого хранилища.

**-v <номер версии хранилища>** — номер версии хранилища, для которого нужно установить метку. Если версия не указана, метка устанавливается для самой последней версии хранилища. Если указана несуществующая версия, будет выдана ошибка.

**-name"<название метки>"** — название метки в двойных кавычках.

**-comment "<текст комментария>" **— комментарий метки в двойных кавычках. Чтобы установить многострочный комментарий, для каждой строки следует использовать свой параметр **comment**.
