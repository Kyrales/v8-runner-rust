# DumpDBCfg

- Вид: `command`
- Раздел: Конфигурация и расширения
- Путь в оглавлении: Конфигурация и расширения -> DumpDBCfg
- Source pagePath: `zif3_dumpdbcfg`
- Source URL: `http://localhost:8080/ru/1%D0%A1%3A%D0%9F%D1%80%D0%B5%D0%B4%D0%BF%D1%80%D0%B8%D1%8F%D1%82%D0%B8%D0%B5?page=zif3_dumpdbcfg`

## Синтаксис

```text
/DumpDBCfg <имя cf-файла> [-Extension <имя расширения>]
```

## Нормализованное описание

— сохранение конфигурации базы данных в файл. Доступные параметры:

**Extension <Имя расширения>** — обработка расширения с указанным именем. Если расширение успешно обработано возвращает код возврата 0, в противном случае (если расширение с указанным именем не существует или в процессе работы произошли ошибки) — 1.

**Внимание!** Если в параметре командной строки требуется ввести имя файла, то следует учитывать, что при указании имени файла с полным путем все каталоги, входящие в состав пути, должны существовать.
