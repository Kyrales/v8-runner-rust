## Синтаксис

```text
/CreateDistributionFiles [-cffile <имя cf-файла>]
[-cfufile <имя cfu-файла> [-f <имя cf-файла>|-v <версия дистрибутива>]+]
[-digisign <имя файла с параметрами лицензирования >][-WarningsAsErrors]
```

## Нормализованное описание

— создание файлов поставки и обновления.

Перед исполнением проверяется соответствие подписи мобильного клиента (если она существует) актуальным метаданным. При несоответствии процесс не прерывается, а в файл вывода служебных сообщений (указывается ключом **/Out**) выводится предупреждение.

Доступные параметры:

**-cffile <имя cf-файла>** — указание создать дистрибутив;

**-cfufile <имя cfu-файла>** — указание создать обновление;

**-f <имя cf-файла>** — дистрибутив, включаемый в обновление, задан именем;

**-v <версия дистрибутива>** — дистрибутив, включаемый в обновление, задан версией;

**-digisign <имя файла с параметрами лицензирования>** — указание параметров лицензирования рабочего места пользователя;

**-WarningsAsErrors** — если указан, то несоответствие подписи и метаданных будет трактоваться, как ошибка, процесс создания файлов поставки и обновления прерывается.

**Примечание**: группа параметров **-f <имя cf-файла>|-v <версия дистрибутива>** повторяется столько раз, сколько файлов дистрибутивов включается в обновление.

**Внимание!** Если в параметре командной строки требуется ввести имя файла, то следует учитывать, что при указании имени файла с полным путем все каталоги, входящие в состав пути, должны существовать.
